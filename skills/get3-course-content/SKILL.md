# 得到课程内容与结构设计

基于《得到品控手册》的课程研发方法论，将课程单课内容从"知识堆砌"升级为"认知模型交付"，让用户获得真正的"Aha时刻"。

## 核心能力：课程内容与结构设计

### 一、建立认知模型

**核心理念**：学习课程不是让用户掌握新知识，而是让用户掌握新认知。最重要的方式是"建立模型"。

#### 认知模型建立方法

1. **从"人"而非"概念"开始**
   - 要快速了解一个领域，不是从概念开始，而是先抓学术大师、关键人物
   - 通过"人"这个起始点，抓住课题的当事人，就是抓住了行动的主体
   - 示例：了解心理学，从弗洛伊德、斯金纳、马斯洛、皮亚杰等人入手

2. **认知模型改变的目标**
   - 课程交付就是用户的认知模型发生了改变，让新知识的涌入成为可能
   - 当用户对同一堆事实重新建模时，就是学习的快感喷涌而出的"Aha时刻"

3. **模型化交付标准**
   - 用户能否用新的认知框架重新理解已知事实
   - 用户能否将模型迁移应用到新场景中

---

### 二、挑战-解决方案结构

**核心理念**：得到课程的核心结构是"挑战-解决方案"。世界不是由领域构成的，世界是由挑战构成的。

#### 传统讲法 vs 挑战-解决方案讲法

| 维度 | 传统讲法 | 挑战-解决方案讲法 |
|------|----------|-------------------|
| **出发点** | 按领域分类（政治、经济、文化） | 从真实挑战出发 |
| **内容组织** | 数据和概念的堆砌 | 问题驱动，有明确的解决目标 |
| **学习体验** | 有挂一漏万之憾 | 在解决挑战中获得认知升级 |

#### 经典案例对比

**介绍北京的课程设计**：

- **传统讲法**：按政治、经济、文化、历史、气候、物产、商业等领域介绍
- **挑战-解决方案讲法**："假设一个高中生，暑假要到北京玩两天，有500元预算，他希望对北京城的了解最大化，请你替他设计一个包括食宿游购娱各种因素的旅行计划，并解释为什么。"

### 三、课程标题设计

**核心理念**：课程表必须经过设计，每一节课的标题一般以"核心任务""挑战""认知交付"这三者选二构成。

#### 标题构成模式

| 模式 | 构成 | 示例 |
|------|------|------|
| **核心任务+挑战** | 功能 + 如何实现 | 《02·功能：心脏如何实现精准供血》 |
| **认知交付+挑战** | 概念 + 如何提高 | 《02·共生：微生物如何提高我们的生存优势》 |
| **核心任务+认知交付** | 功能 + 本质 | 《03·机制：血压调节的底层逻辑》 |

#### 标题设计检查

1. 标题是否包含"核心任务""挑战""认知交付"中的至少两个要素？
2. 标题是否让用户一眼看出这节课要解决什么问题？
3. 标题是否具有足够的信息量，同时保持简洁？

### 四、素材策展

**核心理念**：使用素材本质上是"策展"——片段选择、赋予意义、制造流程、塑造感受、定义边界。

#### 策展人思维

- **不是农夫式**：守着一颗种子变大
- **而是探险家式**：在月球上只切出一块岩石，带回来给地球人看

#### 素材策展五要素

| 要素 | 含义 | 操作要点 |
|------|------|----------|
| **片段选择** | 从海量素材中精选关键片段 | 选择最具代表性的案例和场景 |
| **赋予意义** | 为素材注入认知价值 | 每个素材都服务于认知模型构建 |
| **制造流程** | 设计素材呈现的顺序和节奏 | 由浅入深，由具体到抽象 |
| **塑造感受** | 通过素材调动用户情感 | 让用户产生共鸣和代入感 |
| **定义边界** | 明确素材的适用范围和局限 | 避免过度泛化 |

#### 感性材料运用原则

> 在概念的大海上，用户渴求感性材料，就像迁徙的鸟在无边的洋面上寻找落脚的岛屿。

- 能用几个人、几个场景、几个瞬间讲清楚的事，就不要用抽象概念和逻辑
- 感性材料是概念海洋中的"岛屿"，让用户有地方落脚
- 每个感性材料都应该服务于认知模型的建立

### 五、课程结构设计的完整框架

**核心理念**：课程设计的逻辑原点是让用户获得对新领域的"掌控感"，建立"感性路标"。

#### 标准结构模板

```
挑战
+ 认知起点（原来的解决方法）
+ 认知终点（更好的解决方法）
+ 感性素材
+ 结论（一个模型）
+ 这个结论的其他应用场景
+ 一个行动建议
```

#### 结构要素详解

| 要素 | 作用 | 设计要点 |
|------|------|----------|
| **挑战** | 建立问题意识 | 必须是用户真实面临的挑战 |
| **认知起点** | 呈现用户现有解决方案 | 让用户产生"这就是我"的认同感 |
| **认知终点** | 展示更好的解决方案 | 形成认知落差，激发学习动机 |
| **感性素材** | 提供具象化的理解支撑 | 人、场景、瞬间 > 抽象概念 |
| **结论（模型）** | 交付可迁移的认知框架 | 用户能带走的核心成果 |
| **应用场景** | 展示模型的延展性 | 让用户看到更多可能性 |
| **行动建议** | 促进行为转化 | 具体、可执行、有明确收益 |

#### 实际应用案例

- **《汪志谦·MOT体验设计课》**：能帮助用户做取舍，设立目标，设计好人生中的关键时刻
- **商业案例分析**：分析商业案例成功或失败背后的根本原因，用课程中案例的方法能解决的最普遍的问题是什么

### 六、Aha时刻设计

**核心理念**：Aha时刻是认知模型发生改变的瞬间，是课程交付的核心标志。

#### Aha时刻的特征

1. **重新建模**：用户对同一堆事实产生了新的理解方式
2. **认知落差**：从"原来如此"到"竟然可以这样"的惊喜感
3. **可迁移性**：获得的认知模型可以应用到其他场景

#### Aha时刻的设计方法

1. **先建立旧认知**：让用户清楚自己原来的理解方式
2. **呈现反直觉事实**：展示与旧认知冲突的新信息
3. **提供新模型**：给出能够整合新旧信息的认知框架
4. **验证模型有效性**：用多个案例证明新模型的解释力

---

## 使用方式

根据用户的具体需求，选择对应模块进行设计：

- **认知模型构建**：使用"从人而非概念开始"的方法，建立可迁移的认知框架
- **挑战-解决方案结构设计**：用真实挑战替代领域分类，构建问题驱动的内容
- **课程标题设计**：使用"三者选二"法则，确保标题包含核心任务/挑战/认知交付
- **素材策展**：运用五要素法，精选感性材料作为概念海洋中的"岛屿"
- **完整课程结构设计**：使用标准七要素模板，确保每节课都有完整的认知交付
- **Aha时刻设计**：通过"旧认知→反直觉事实→新模型→验证"四步法，创造认知突破
